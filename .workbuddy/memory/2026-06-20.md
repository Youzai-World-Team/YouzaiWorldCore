# 2026-06-20 工作日志

## 修复 Minecraft 26.2 账户系统编译错误 (第二轮)

修复了剩余 11 个编译错误，根因均为 Minecraft 26.2 / authlib 9.0.75 API 变更。

### 第二轮修改的文件

1. **AccountEventHandler.java** — `GameProfile.getName()` → `GameProfile.name()` (authlib 9.0.75 中 GameProfile 改为 Java Record)
2. **ModNetworking.java** — `context.player().getServer()` → `context.player().level().getServer()`
3. **ClientNetworking.java** — `GameProfile.getName()` → `.name()`；`client.screen` 不可用，改用静态 `currentAccountScreen` 字段跟踪
4. **YouzaiworldCore.java** — `GameProfile.getName()` → `.name()` (2处)；`getPlayerOrException()` 方法签名添加 `throws CommandSyntaxException` (5个方法)

### 关键 API 变更
- `com.mojang.authlib.GameProfile` 现在是 Java `Record`，方法名从 `getName()` 变更为 `name()`
- `Minecraft.screen` 在 26.2 中为 private，不再可公开访问
- `source.getPlayerOrException()` 需要 `throws CommandSyntaxException`
